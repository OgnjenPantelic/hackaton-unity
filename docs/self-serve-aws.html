AWS Self-serve Deployments

Emphasizes speed of deployment over flexibility of customization. Useful if the customer prefers a more Databricks-managed approach to workspace creation and maintenance.

Express Deployment

  • Creates a new Databricks Account and starts the free trial
  • No resources deployed in the customer's AWS account (no cloud permissions required)
  • Fully managed SaaS experience (Serverless compute with default cloud storage)
  • Does not support Classic compute
  • Ideal for first-time customers with no DevOps/Infra team
  • Ideal for quick exploration on Databricks with minimal setup

Serverless Deployment

  • Requires an existing Databricks account
  • No resources deployed in the customer's AWS account (no cloud permissions required)
  • Fully managed SaaS experience (Serverless compute with default cloud storage)
  • Does not support Classic compute
  • Ideal for first-time customers with no DevOps/Infra team
  • Ideal for quick exploration on Databricks with minimal setup

Marketplace Deployment

  • Creates a new Databricks Account and starts the free trial
  • Requires AWS cloud permissions as resources (S3, IAM, VPC) are deployed in the customer's AWS account
  • Ideal for first-time customers as well as existing customers without stringent DevOps/Infra practices
  • Ideal for POC or quick use case exploration
  • Supports both serverless and classic compute options

Classic workspace (Automated configuration)

  • Requires an existing Databricks account
  • Requires cloud permissions, as an AWS admin must grant Databricks temporary access to provision resources (S3, VPC, IAM) in the AWS account
  • Ideal for first-time customers as well as existing customers without stringent DevOps/Infra practices
  • Ideal for POC or quick use case exploration
  • Supports both serverless and classic compute options
