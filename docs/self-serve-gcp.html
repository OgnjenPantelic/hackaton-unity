GCP Self-serve Deployments

Emphasizes speed of deployment over flexibility of customization. Useful if the customer prefers a more Databricks-managed approach to workspace creation and maintenance.

Serverless Workspace

  • No resources deployed in the customer's GCP Project (no cloud permissions required)
  • Ideal for first-time customers with no DevOps/Infra team
  • Ideal for quick exploration on Databricks with minimal setup
  • Fully managed SaaS experience (Serverless compute + Default Storage)
  • Does not support Classic compute

Databricks-Managed VPC Deployment

  • VPC, GCS & IAM created and managed by Databricks in the customer's GCP project
  • Requires GCP cloud permissions
  • Ideal for first-time customers as well as existing customers without stringent DevOps/Infra practices
  • Ideal for POCs and quick explorations
  • Supports both serverless and classic compute
